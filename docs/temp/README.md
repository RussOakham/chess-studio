# Temporary planning files (`docs/temp/`)

Markdown here is **local / work-in-progress** (implementation checklists, spikes, and archived specs). It may be **gitignored** by `*.temp.*` patterns in some setups—confirm with `git check-ignore -v <file>` if a file does not appear in commits.

**Canonical docs to use instead:**

- Game feature status: [`docs/implementation/game-implementation-plan.md`](../implementation/game-implementation-plan.md)
- Convex + auth + env: [`docs/planning/convex-auth-data.md`](../planning/convex-auth-data.md)
- Game review / Lichess / QA: [`docs/game-review-cross-cutting-qa.md`](../game-review-cross-cutting-qa.md)

When a temp doc’s work is done, prefer **updating the canonical doc** and adding a short “Archived” line at the top of the temp file (or delete the temp file if fully redundant).
